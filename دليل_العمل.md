# 💧 نظام تحليل شبكات المياه — دليل العمل الكامل

## 1. نظرة عامة على النظام

النظام مكون من **تطبيقين متكاملين** يعملان معاً:

| التطبيق | الوظيفة | المنفذ |
|---------|---------|--------|
| **التطبيق الرئيسي (React + FastAPI)** | محاكاة هيدروليكية + خريطة تفاعلية | `3000` (واجهة) + `8002` (باكند) |
| **التطبيق الميداني (Streamlit)** | تحليل الطلب التراكمي + كشف الفاقد ومناطق الاشتباه | `8501` |

### هيكل المجلدات

```
الخميس مساء/
├── backend/              ← خادم FastAPI
│   └── app/main.py       ← API endpoints (محاكاة + إدارة Streamlit + مشاركة البيانات)
├── frontend/             ← تطبيق React (خريطة + محاكاة)
│   └── src/
│       ├── App.tsx             ← الصفحة الرئيسية
│       └── components/
│           ├── MapView.tsx     ← خريطة Leaflet
│           ├── StreamlitButton.tsx  ← زر فتح التحليل المتقدم
│           └── ...
├── run_all.bat           ← تشغيل الخادمين معاً
└── repo/ (GitHub — مستودع منفصل)
    ├── app.py            ← تطبيق Streamlit (تحليل ميداني)
    ├── core_analysis.py  ← محرك التحليل
    └── requirements.txt
```

## 2. تدفق البيانات بين التطبيقين

```
┌─────────────────────┐      POST /save-subscribers      ┌──────────────┐
│                     │ ──────────────────────────────►   │              │
│   React App (3000)  │                                   │  FastAPI     │
│   (محاكاة + خريطة)   │                                   │  Backend     │
│                     │ ◄──────────────────────────────   │  (port 8002) │
└─────────────────────┘      GET /subscribers              │              │
                                                           │              │
┌─────────────────────┐      GET /subscribers              │              │
│                     │ ──────────────────────────────►   │              │
│  Streamlit App      │                                   │              │
│  (تحليل, port 8501)  │                                   └──────┬───────┘
│                     │                                          │
│                     │   POST /streamlit/start                  │
│  ← يفتح عبر زر في   │   GET  /streamlit/status                 │
│     React App       │ ◄───────────────────────────────────     │
└─────────────────────┘                                          │
                                                         ┌──────┴───────┐
                                                         │ subprocess   │
                                                         │ تشغيل Streamlit│
                                                         └──────────────┘
```

### آلية الربط خطوة بخطوة:

1. **React App ← FastAPI**: عند تحميل/تعديل بيانات المشتركين في التطبيق الرئيسي، `useEffect` في `App.tsx` يستدعي `postSubscribers()` التي ترسل الـ JSON إلى `POST /save-subscribers`.

2. **Streamlit ← FastAPI**: عند الضغط على زر "تحميل من التطبيق الرئيسي" في Streamlit، تستدعي `fetch_subscribers_from_main_app()` العنوان `GET /subscribers` وتُحول الإحداثيات إلى `geometry`.

3. **React → تشغيل Streamlit**: زر `StreamlitButton` يتحقق من حالة Streamlit عبر `GET /streamlit/status`، وإذا كان متوقفاً يشغله عبر `POST /streamlit/start`.

## 3. كيف يعمل التطبيق الرئيسي (React)

### الوظائف:
- **رفع Excel**: استيراد بيانات المشتركين (الاسم، الارتفاع، الطلب، Qmax، الإحداثيات)
- **خريطة تفاعلية (Leaflet)**: عرض المشتركين والشبكة والأحياء
- **رسم شبكة المياه**: خطوط ✏️ و🔲 أحياء
- **ربط المشتركين**: تلقائي أو يدوي بأقرب خط مياه
- **جلب الارتفاعات**: من API خارجي
- **محاكاة هيدروليكية**: تشغيل WebSocket لمحاكاة توزيع المياه زمنياً
- **تحكم أثناء المحاكاة**: إيقاف/تشغيل الضخ، تعديل Q_in
- **تحليل ذكي (ولوحة تحليل)**: تصنيف حسب ΔH، منحنيات WI
- **حفظ/فتح المشاريع**: بصيغة `.wsm`

### Endpoints في الباكند:

| المسار | الطريقة | الوظيفة |
|--------|---------|---------|
| `/subscribers` | GET | إرجاع آخر بيانات المشتركين (لتطبيق Streamlit) |
| `/save-subscribers` | POST | حفظ بيانات المشتركين من React |
| `/upload-excel/` | POST | رفع Excel وتحليله |
| `/ws/{simulation_id}` | WebSocket | محاكاة هيدروليكية زمنية |
| `/streamlit/status` | GET | هل Streamlit يعمل؟ |
| `/streamlit/start` | POST | تشغيل Streamlit في الخلفية |
| `/streamlit/stop` | POST | إيقاف Streamlit |
| `/streamlit/open` | GET | فتح Streamlit (مع تشغيله إن لزم) |

## 4. كيف يعمل التطبيق الميداني (Streamlit)

### نظرية التحليل — منحنى الطلب التراكمي:

**الفكرة الأساسية**: إذا كان لديك `N` مشترك وكمية مياه `Q`، فالمتوقع أن تصل المياه إلى المشتركين ذوي المنسوب الأقل أولاً.

```
خطوات التحليل:
1. احسب ΔH لكل مشترك = ارتفاع المشترك - أدنى ارتفاع في المجموعة
2. احسب الطلب الموزون = 0.5×LastMonth + 0.3×Avg3Months + 0.2×Avg12Months
3. رتّب المشتركين تصاعدياً حسب ΔH
4. احسب الطلب التراكمي التصاعدي
5. ارسم منحنى (طلب تراكمي) مقابل (ΔH)
6. أسقط Q على المنحنى ← ΔH النظري (أقصى ارتفاع تصل إليه المياه)
7. كل مشترك ΔH < ΔH النظري ← "متوقع وصول" والعكس "غير متوقع"
8. قارن ΔH النظري بالفعلي من الميدان
```

### قيم المدخلات في Streamlit:

| الحقل | الوصف |
|-------|-------|
| **KML/KMZ** | نقاط المشتركين من GIS (Points ← مشتركين، Polygons ← أحياء) |
| **Excel استهلاك** | `CustomerID, LastMonth, Avg3Months, Avg12Months` |
| **فروق المنسوب ΔH** | إما قيمة افتراضية واحدة، أو رفع CSV لكل مشترك |
| **كمية الضخ Q (m³)** | إجمالي المياه المضخوخة في الدفعة |
| **ΔH الوصول الفعلي** | إما ΔH مباشر أو اختيار "آخر مشترك وصل" |
| **الحالة الفعلية (CSV)** | اختياري — `CustomerID, Status` (وصلت/لم تصل/ضغط ضعيف) للمقارنة |

### المخرجات:

| المخرج | الوصف |
|--------|-------|
| **المشتركين المخدومين** | قائمة بمن يتوقع وصول المياه لهم |
| **ΔH النظري** | أقصى ΔH تصل إليه المياه بناءً على Q |
| **مؤشرات الأداء** | فاقد (%)، درجة اشتباه (%)، مؤشر وصول (%) |
| **مناطق الاشتباه** | فجوات تغطية بين المشتركين الذين وصلت لهم المياه |
| **منحنى الطلب التراكمي** | رسم بياني يوضح Q، الوصول النظري والفعلي |
| **مقارنة بالواقع** | مصفوفة (True Positive, False Negative, ...) إذا وفّرت بيانات الفعلي |

## 5. مثال تطبيقي كامل (Walkthrough)

### السيناريو:
حي سكني فيه 50 مشتركاً، كمية الضخ `Q = 5000 m³`، نريد معرفة من يصلهم الماء وأين يوجد فاقد أو اشتباه.

### باستخدام التطبيق الرئيسي (React):

```
1. افتح http://localhost:3000
2. ارفع ملف Excel بمشتركي الحي (أو استخدم "معاينة" لتوليد 50 عشوائي)
3. تظهر النقاط على خريطة Leaflet
4. شغّل الباكند (run_all.bat أو شغل_الباكند.bat)
5. اضبط معاملات المحاكاة وابدأ — سترى وصول المياه زمنياً
6. الآن بيانات المشتركين موجودة في الباكند (GET /subscribers)
```

### باستخدام التطبيق الميداني (Streamlit):

```
طريقة 1 — من زر React:
  7. في تطبيق React، اضغط "📈 التحليل المتقدم (الطلب التراكمي)"
  8. يفتح Streamlit تلقائياً على http://localhost:8501

طريقة 2 — مباشر:
  7. افتح http://localhost:8501 يدوياً
  8. في الشريط الجانبي، اضغط "📥 تحميل من التطبيق الرئيسي"
  9. تُستورد بيانات المشتركين الـ 50 تلقائياً

بعد فتح Streamlit:
  10. إذا لم تحمّل من API، ارفع KML للمشتركين وExcel للاستهلاك
  11. أدخل Q = 5000 في حقل "كمية الضخ"
  12. أدخل ΔH الفعلي — مثلاً أدخل 25 متر كقيمة افتراضية
  13. اضغط "تشغيل التحليل"

النتائج:
  - يظهر جدول المشتركين المخدومين (مثلاً 38 من 50)
  - ΔH النظري = 32.4 متر
  - مؤشر الوصول = 76% (أي 38/50)
  - الفاقد المقدر: فرق Q بين △H النظري والفعلي
  - مناطق الاشتباه: تظهر على الخريطة كنقاط حمراء
  - منحنى الطلب التراكمي يوضح العلاقة بين Q والارتفاع
```

### تفسير النتائج للمثال:

| المؤشر | القيمة | التفسير |
|--------|--------|---------|
| عدد المشتركين | 50 | |
| Q | 5000 m³ | |
| ΔH النظري | 32.4 m | إذا ΔH فعلي > 32.4، المياه لا تصل |
| المخدومين | 38 | من الـ 50 |
| مؤشر الوصول | 76% | 38/50 |
| الفاقد | ~12% | الفرق بين النظري والفعلي |
| مناطق الاشتباه | 3 فجوات | فيها احتمال فاقد أو سرقة أو عطل |

## 6. إعادة التعريفات الأساسية

```
ΔH = ارتفاع المشترك - أدنى ارتفاع في الحي (m)
الطلب الموزون = 0.5 × LastMonth + 0.3 × Avg3Months + 0.2 × Avg12Months
Q = كمية الضخ (m³) — إجمالي المياه المضخوخة في الدفعة
ΔH النظري = نقطة تقاطع Q مع منحنى الطلب التراكمي
الوصول النظري = المشتركين ذوي ΔH < ΔH النظري
الفجوة = عدد المشتركين المتوقع وصولهم - عدد من وصل فعلاً
مؤشر الوصول = المخدومين / الكل
درجة الاشتباه = 0.4×(فجوة التغطية) + 0.3×(الفقد) + 0.3×(1-مؤشر الوصول)
```

## 7. التشغيل

### يدوياً:

```bash
# Terminal 1 — الباكند
cd backend
python -m uvicorn app.main:app --reload --port 8002

# Terminal 2 — التطبيق الميداني
cd repo  # أو المسار الذي فيه app.py
streamlit run app.py --server.port 8501

# Terminal 3 — تطبيق React
cd frontend
npm start
```

### بنقرة واحدة (Windows):

شغّل `run_all.bat` — يشغل FastAPI + Streamlit تلقائياً في نافذتين منفصلتين.
