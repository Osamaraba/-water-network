# 💧 نظام تحليل ضخ وتوزيع المياه — ميداني

تطبيق **Streamlit** لتحليل وصول المياه في شبكات التوزيع باستخدام بيانات:
- **GIS** (KML) — مواقع المشتركين
- **استهلاك شهري** (Excel) — استهلاك كل مشترك
- **فروق المنسوب** (ΔH) — فرق ارتفاع المشترك عن أدنى نقطة
- **كمية الضخ** (Q) — كمية المياه المضخوخة في الدفعة

> 📖 للشرح الكامل عن النظام (بما في ذلك التكامل مع تطبيق React للمحاكاة الهيدروليكية ومثال تطبيقي خطوة بخطوة)، راجع **[دليل_العمل.md](دليل_العمل.md)**.

## 📦 المتطلبات

```bash
pip install -r requirements.txt
```

## 🚀 التشغيل

```bash
streamlit run app.py
```

يفتح تلقائياً على `http://localhost:8501`

## 📂 تنسيق الملفات

### ملف KML (المشتركين)
- Points (نقاط) تمثل المشتركين
- يُضاف `CustomerID` تلقائياً إن لم يوجد

### ملف Excel (الاستهلاك)
| CustomerID | LastMonth | Avg3Months | Avg12Months |
|------------|-----------|------------|-------------|
| 1          | 250       | 230        | 210         |
| 2          | 180       | 190        | 185         |

### ملف CSV (فروق المنسوب ΔH) — اختياري
| CustomerID | delta_h |
|------------|---------|
| 1          | 12.5    |
| 2          | 8.3     |

### ملف CSV (الحالة الفعلية) — اختياري للمقارنة
| CustomerID | status    |
|------------|-----------|
| 1          | وصلت      |
| 2          | لم تصل    |
| 3          | ضغط ضعيف  |

## 📊 المخرجات

- **مؤشرات الأداء**: الوصول النظري/الفعلي، الفاقد، درجة الاشتباه
- **منحنى الطلب التراكمي** — موقع Q على المنحنى
- **مناطق الاشتباه** — فجوات التغطية
- **مقارنة بالواقع** — إذا وفّرت بيانات الحالة الفعلية
- **تصدير CSV** وحفظ الجلسة

## 🧠 الفكرة الأساسية

1. ترتيب المشتركين تصاعدياً حسب ΔH
2. حساب الطلب التراكمي
3. إسقاط Q على المنحنى ← ΔH النظري
4. مقارنة ΔH النظري بالفعلي من الميدان ← مؤشرات الأداء
5. اكتشاف مناطق الاشتباه (فاقد/سرقة) من الانحراف بين النظري والواقعي
